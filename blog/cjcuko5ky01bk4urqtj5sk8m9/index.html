浅谈 B- 感受

发表于 2010-08-17

     本公司有着这样一个规定：每月，老大和上司们都要根据你的综合情况给出 A - C 不等的评价。对于不同的等级，每月的

月绩效和年绩效乃至以后的发展都会受到一定的影响。


    一直被评为 B 的我，每月都在不停的努力，期望能够在本月得到一个 A 或 B+，可惜徒劳无功，直到上个月得了个 B-。



原来“力”使“错”了。每个人的世界观、价值观有所不同，对事物乃至软件工程的看法也有一定的差别。你与上司的立场不同，


职责不同，当然想法深度也就会有所不同了。理解也就会有一定的偏差。

     上月依旧在做前端插件的开发，听着老大对插件的构想与憧憬。在完成任务的时候进行了多方面的考虑。前期花了不少时

间在框架的搭建上面。等框架稳定了，便根据老大所传达的大略提示开始功能的制定和完善。在插件扩展为2个的时候，还为其

取名为 bowknot。由于公司每次列会都会谈到权利责任的问题，什么事情你有权利自己确定，什么事情没有。面对给项目取名

这件事情，我是这么看待的：自己在这个插件里面倾入了很多心血，更能懂其意义，名其定位之所在。你和老大提出取名事项


，老大肯定没时间去想这个，任务最终还是到你头上。如果你想了个名字，老大提出更好的，随时可以更改。就好像 Java 以

前并不叫 Java 一样，没什么的。其实这个权利有时候真的很难把握，少问一点，导致沟通不够；多问一点，老大太忙，也不

忍心打扰，可能还会被说没技术没主见。。。也就是因为这个权利的问题，老大说那里要改，哪里要怎么怎么。自己明白后，

再和老大确认后。自己就下去 Hi 起了。弄好后也就发布到公司的论坛中了（公司有一个论坛，各种文档都往上面发，

还可以在上面写论坛式的 blog - -！）。老大分配的任务做完后，发现一些 bug，想到一些改进，也都一并发到论坛中了。在

没接到 B- 时，还觉得颇有成效，偶尔也小乐一下 ^^

    当然，以上说了那么大一堆，都是从我的角度出发 :）但是从老大的立场角度出发，“在工作成果提交过程中，较为随意

，工作中主动沟通交流不够。”这个为 B- 的评语也完全正确。

    面对此情此景，我辛辛苦苦、勤勤恳恳，我错了嘛？老大中肯的评语错了嘛？如果老大在发现我哪里不足，哪里有问题的

时候，抽个10秒（以上评语说完大概不要5秒吧 ^^）和我说一下或 qq 上发一下（公司上班必须挂 QQ，用以交流，否则要扣工资

–！），也许结果完全就不一样了。老大再忙，这么重要，这么带有建议性、指导性的10秒应该是可以挤的出来的吧？我觉得员工

并不是腐朽到顽固不化（那样的员工公司也不会要），只是有时候一心想着把任务完成的更好，往往在某些方面忽略了。这时候

，也许一句话就能点醒，使其做的比以前更好，而不是一直维持着不足。就像本周还未接到评论之前，我还在努力的考虑着应

该怎么完善，怎么改进。提前一点指点，提前一点告知，也许我们能做的更好，而不是去抱怨自己如何努力却换来了一个 B-。

我相信以指导、提点、教导的方式去替代最终辛辛苦苦换来的 B-，更能为公司培养出优秀、热情、充满活力的员工。

     最后衷心希望公司能够广纳到更多的人才，培养出更多优秀的人才。(最近看三国，历史和电视都这么说的 ^^)
